# Treeniapp (Valkku) — Tekoälyvalmentaja

**Päivämäärä:** 2026-07-14
**Laajuus:** Uusi ominaisuus. Keskusteleva tekoälyvalmentaja, joka lukee sovelluksen olemassa olevaa dataa (Sali, Aerobinen, Keho, Uni, Ruoka) ja antaa perusteltuja ehdotuksia — ei koskaan muuta dataa itse, vain ehdottaa.
**Riippuvuudet:** Uusi Supabase Edge Function, uusi tietokantataulu (`coach_messages`), käyttäjän oma Claude API -avain (Anthropic), uudet Supabase-secretit (samaan tapaan kuin `VAPID_PRIVATE_KEY`/`CRON_SECRET`).

---

## Tausta ja tavoite

Käyttäjä halusi tekoälyavusteisen valmentajan, joka antaa ehdotuksia oman datan pohjalta ja "oppii" ajan myötä mikä toimii käyttäjälle — mutta ei koskaan tee muutoksia itsenäisesti (esim. ei muokkaa harjoitusohjelmaa tai kirjaa ruokaa puolestasi). "Oppiminen" toteutetaan syöttämällä valmentajalle tuore yhteenveto oikeasta historiadatasta jokaisen viestin yhteydessä — ei mallin varsinaista kouluttamista, joka olisi ylimitoitettu yhden käyttäjän muutaman kuukauden dataan.

V1 rajaa pysyvän "muistiinpano"-mekanismin (valmentaja kirjoittaisi itse havaintoja käyttäjästä ajan myötä) pois — se on looginen v2-jatko kun tämä perusversio on osoittautunut hyödylliseksi.

---

## 1. Interaktiomalli

Keskusteleva chat, ei automaattisesti generoituvia yhteenvetoja. Käyttäjä avaa Valmentaja-sivun, aloittaa keskustelun tai jatkaa vanhaa, ja kysyy mitä haluaa ("miten tämä kuukausi on mennyt?", "pitäisikö nostaa penkkipunnerruksen painoa?").

---

## 2. Arkkitehtuuri

Selain ei voi kutsua Claude APIa suoraan — API-avain paljastuisi sivun JavaScriptistä. Ratkaisu on sama malli kuin olemassa olevassa `check-and-notify`-Edge Functionissa:

1. Selain lähettää käyttäjän viestin + `conversation_id`:n uudelle `coach-chat`-Edge Functionille.
2. Function todentaa pyynnön (ks. Turvallisuus alla), hakee tuoreen datayhteenvedon Supabasesta (service-role-avaimella), rakentaa koko promptin (ohjeistus + datayhteenveto + keskusteluhistoria + uusi viesti) ja kutsuu Claude APIa käyttäen secrettinä tallennettua avainta.
3. Function palauttaa vastauksen selaimelle, joka tallentaa sekä käyttäjän viestin että vastauksen `coach_messages`-tauluun ja näyttää ne.

**Deploy:** `coach-chat` deployataan JWT-todennettuna (TOISIN kuin `check-and-notify`, joka käyttää `--no-verify-jwt` + oma cron-secret) — eli tavallinen Supabase-anon-key-todennus vaaditaan, sama kuin jokaisella muulla taulukutsulla sovelluksessa.

### Turvallisuus

Sovelluksessa ei ole kirjautumista — kaikki taulut käyttävät sallivaa RLS:ää (`using (true)`) anon-avaimen takana, joka on jo julkinen (`index.html`:n lähdekoodissa). Tämä on tietoinen, aiemmin hyväksytty riski yhden käyttäjän sovellukselle. Tekoälyominaisuus tuo kuitenkin uuden riskin: toisin kuin tietokantakirjoitus, jokainen väärinkäyttökutsu maksaisi oikeaa rahaa (Claude API -tokenit). Siksi kaksi lisäsuojausta, joita muualla sovelluksessa ei tarvita:

1. **Salasanaportti:** `coach-chat` tarkistaa `x-coach-secret`-headerin Supabase-secrettiä vastaan (sama malli kuin `CRON_SECRET`). Valmentaja-sivu kysyy salasanan kerran, tallentaa sen `localStorage`aan ja lähettää sen jokaisen pyynnön mukana. Tämä nostaa kynnystä yli pelkän julkisen anon-avaimen tuntemisen.
2. **Päivittäinen viestilaskuri:** Function laskee tämän päivän `coach_messages`-rivit (`role='user'`) ennen Claude-kutsua; rajan (esim. 100/pv) ylittyessä palautetaan virhe eikä Claude APIa kutsuta. Sulkee pois skenaarion jossa joku löytää endpointin ja ajaa laskun ylös.

Claude API -avain itse on Supabase-secret, käytössä vain Edge Functionin sisällä — ei koskaan selaimessa.

---

## 3. Tietokanta

Uusi migraatio:

```sql
create table coach_messages (
  id              bigint generated by default as identity primary key,
  conversation_id uuid not null,
  role            text not null check (role in ('user','assistant')),
  content         text not null,
  created_at      timestamptz not null default now()
);

alter table coach_messages enable row level security;

create policy coach_messages_select on coach_messages
  for select to anon, authenticated using (true);
create policy coach_messages_insert on coach_messages
  for insert to anon, authenticated with check (true);
```

`conversation_id` ryhmittelee viestit yhdeksi keskusteluksi — generoidaan selaimessa uutena UUID:na kun käyttäjä aloittaa uuden keskustelun. Ei erillistä "conversations"-taulua: keskustelulista rakennetaan ryhmittelemällä `coach_messages` `conversation_id`:n mukaan, näyttäen ensimmäisen käyttäjäviestin otsikkona ja tuoreimman aikaleiman. Ei update/delete-policya — historia on append-only, sama periaate kuin `user_profile`ssa.

## 4. Datayhteenveto valmentajalle

Jokaisen viestin yhteydessä Edge Function rakentaa tuoreen yhteenvedon (ei koko raakahistoriaa — kallista ja vaikeaa mallille tulkita):

- **Profiili:** sukupuoli, pituus, ikä — pohja BMR/kaloripäättelylle.
- **Viikkoyhteenvedot, viim. ~12 viikkoa:** yksi rivi per viikko — salikäynnit, aktiviteettimäärä/tyypit, kilometrit, unen keskiarvo + syvä/REM, paino jos mitattu, päivän nettokalorien keskiarvo. Riittää trendien havaitsemiseen ilman jokaisen rivin tyhjentämistä.
- **Tuore erittely, viim. ~2 viikkoa:** oikeat treenisessiot (liikkeet/painot/toistot), päivittäiset ruokasummat, yksittäiset unikirjaukset, yksittäiset aktiviteetit — jotta täsmälliset kysymykset ("miten penkki on kehittynyt") saavat oikean pohjan, ei vain viikkokeskiarvoja.
- **Olemassa olevat lasketut signaalit:** plateau-tunnistin ja paras-juoksuviikko-tarkistus ovat jo olemassa ja tuottavat jo hyvän signaalin ilmaiseksi — valmentaja uudelleenkäyttää niiden tulosta sen sijaan että laskisi saman uudestaan.
- **Tämän hetken tilannekuva:** nykyinen streak, päivän nettokalorit — "miten menee juuri nyt" -kysymyksiin.

Sama taulu-/kyselymalli kuin muualla sovelluksessa jo käytetään (viikkotilastot, aktiviteettikyselyt jne.) — Edge Function toistaa saman logiikan palvelinpuolella (Deno), koska se ei voi suoraan uudelleenkäyttää selaimen JS-funktioita, mutta kyselyjen muoto/aikavälit ovat samat kuin jo todistetusti toimivat.

**Laajuusrajaus:** aloitetaan tällä yhteenvedolla; dataa voidaan lisätä myöhemmin jos vastaukset osoittautuvat liian suppeiksi jollain alueella. Ei placeholder-toteutusta puuttuville alueille — kaikki 5 tietolähdettä (Sali, Aerobinen, Keho, Uni, Ruoka) ovat mukana v1:stä lähtien.

## 5. Käyttöliittymä

Uusi sivu sivupalkista (Ohjelman vieressä), samalla `page-header`/takaisin-nuoli-kuviolla kuin muilla sivuilla.

- **Aloitusnäkymä:** lista aiemmista keskusteluista (ensimmäinen viesti otsikkona, tuorein ensin — uudelleenkäyttää olemassa olevaa `hist-item`-listatyyliä), sekä "+ Uusi keskustelu" -painike ylhäällä.
- **Keskustelunäkymä:** viestiketju (käyttäjän ja valmentajan viestit visuaalisesti erotettuina), syöttökenttä + lähetä-painike alareunassa, ja paluu keskustelulistaan.
- **Salasanakysely:** näytetään kerran jos salasanaa ei löydy `localStorage`sta (tai jos pyyntö palautuu "unauthorized") — yksinkertainen salasanakenttä, sama visuaalinen paino kuin muissa asetusmodaaleissa.

---

## 6. Rajaus (ei tässä versiossa)

- **Pysyvä muistiinpano-mekanismi** (valmentaja kirjoittaisi/päivittäisi havaintoja käyttäjästä ajan myötä erilliseen tauluun) — looginen v2, ei v1:ssä. V1:n "oppiminen" tulee siitä että valmentaja näkee tuoreen datan joka viestillä, ei erillisestä muistista.
- **Valmentaja ei koskaan muuta dataa** — ei kirjoita treenejä, ruokaa, unta tai profiilia puolestasi. Pelkkä luku + ehdotukset tekstinä.
- **Ei automaattisesti generoituvia yhteenvetoja** — pelkkä käyttäjän aloittama chat.

---

## Testaus

Ei automaattitestejä (ei testikehystä projektissa). Manuaalinen läpikäynti:

1. Salasanaportti: pyyntö ilman oikeaa `x-coach-secret`-arvoa palautuu virheenä, ei kutsu Claude APIa.
2. Päivittäinen laskuri: raja-arvon ylitys palauttaa virheen sen sijaan että kutsuisi Claude APIa.
3. Uusi keskustelu aloittaa tuoreen `conversation_id`:n; viestit säilyvät sivun uudelleenlatauksen yli.
4. Keskustelulista näyttää aiemmat keskustelut oikein ja niitä voi jatkaa.
5. Muutaman oikean kysymyksen pistokoe tunnettua dataa vasten (esim. "montako kertaa treenasin viime viikolla?") — varmistaa että vastaukset perustuvat oikeaan dataan, eivät vain uskottavalta kuulostavaan tekstiin.
6. Edge Functionin deploy + secretit (`ANTHROPIC_API_KEY`, `COACH_SECRET`) toimivat päästä päähän.
